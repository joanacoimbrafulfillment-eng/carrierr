📦 Envios em Análise

Cola aqui os IDs em dúvida. Ao guardar, cada ID fica com data de inserção do dia, fica persistido na Firebase em envios_analise/items e o sistema verifica logo se foi PACKING, RETURN ou REFUND. IDs com return/refund passam automaticamente para a lista própria; depois podes marcar como visto para irem para Verificados.
Aceita um ID por linha, ou IDs separados por espaços, vírgulas ou ponto-e-vírgula. Depois de guardar e verificar, esta caixa fica limpa automaticamente.
Data de inserção automática: quando guardas um ID, fica registada a data do dia em que foi inserido. Essa data não muda nas verificações seguintes.
Quando sabes que saiu mas não aparece na Firebase: usa o botão amarelo “Saiu manual” na coluna Ações, ou escreve diretamente na coluna “Saiu manual / nota”. Esse campo fica guardado na Firebase e passa a contar como Embalado = SIM (manual).
Pronto
Os IDs e notas ficam guardados na Firebase. O browser também mantém uma cópia local para emergência, mas a fonte principal é a base de dados.

• active_batches (lotes criados, carrier, data, expected_ids, processed_ids, cancelled_ids, order_details)
• registos (para apanhar tracking e confirmar PACKING)
• Opcional: id_history (modo Profundo)
